## Docker Installation

Docker Compose provides the fastest path to a working QIMEM deployment with Postgres persistence.

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <repository-url>
    cd qimem
    ```
  </Step>

  <Step title="Configure environment variables">
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` with your configuration:
    
    ```bash .env
    QIMEM_MODE=stateful
    DATABASE_URL=postgres://postgres:postgres@postgres:5432/qimem
    QIMEM_BIND=0.0.0.0:8080
    RUST_LOG=info
    ```
    
    <Note>
    For stateless mode (in-memory storage), set `QIMEM_MODE=stateless` and omit `DATABASE_URL`.
    </Note>
  </Step>

  <Step title="Launch with Docker Compose">
    ```bash
    docker compose up --build
    ```
    
    This command:
    - Builds the Rust binaries with release optimizations
    - Starts PostgreSQL 16 with health checks
    - Runs `qimem-api` in stateful mode on port 8080
    - Applies database migrations automatically
    
    <Warning>
    The first build may take 5-10 minutes to compile dependencies.
    </Warning>
  </Step>

  <Step title="Verify the deployment">
    ```bash
    curl -fsS http://localhost:8080/health
    ```
    
    Expected response:
    ```json
    {"status":"ok"}
    ```
    
    Check the ready endpoint:
    ```bash
    curl -fsS http://localhost:8080/ready
    ```
    
    Expected response:
    ```json
    {"status":"ready"}
    ```
  </Step>
</Steps>

### Docker Compose Services

The `docker-compose.yml` defines three services:

<Accordion title="postgres">
  - **Image**: `postgres:16`
  - **Port**: 5432
  - **Database**: `qimem`
  - **Health check**: `pg_isready` every 5 seconds
  - **Environment**:
    - `POSTGRES_USER=postgres`
    - `POSTGRES_PASSWORD=postgres`
    - `POSTGRES_DB=qimem`
</Accordion>

<Accordion title="qimem-api">
  - **Build**: Multi-stage Dockerfile with Rust 1.82
  - **Port**: 8080
  - **Mode**: Stateful with Postgres backend
  - **Features**: `--features stateful`
  - **Health check**: `curl http://localhost:8080/health` every 10 seconds
  - **Dependencies**: Waits for postgres to be healthy
</Accordion>

<Accordion title="devcontainer">
  - **Purpose**: Development environment with Rust toolchain
  - **Usage**: Supports VS Code Remote Containers
  - **Command**: `sleep infinity` (keeps container running)
</Accordion>

## Build from Source

For development or environments without Docker:

<Steps>
  <Step title="Install Rust">
    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    rustup default stable
    ```
    
    Verify installation:
    ```bash
    rustc --version
    cargo --version
    ```
  </Step>

  <Step title="Clone and configure">
    ```bash
    git clone <repository-url>
    cd qimem
    cp .env.example .env
    ```
  </Step>

  <Step title="Run with cargo">
    <CodeGroup>
      ```bash QIMEM API (Stateless)
      QIMEM_MODE=stateless cargo run --bin qimem-api
      ```
      
      ```bash QIMEM API (Stateful)
      # Requires Postgres running
      DATABASE_URL=postgres://postgres:postgres@localhost:5432/qimem \
        QIMEM_MODE=stateful \
        cargo run --bin qimem-api --features stateful
      ```
      
      ```bash QAuth API (Unified Platform)
      # Includes all security + auth endpoints
      cargo run --bin qauth-api
      ```
      
      ```bash QIMEM CLI
      # For local encryption/decryption
      cargo run --bin qimem -- --help
      ```
      
      ```bash QAuth CLI
      # For identity management tasks
      cargo run --bin qauth -- --help
      ```
    </CodeGroup>
  </Step>

  <Step title="Verify the service">
    ```bash
    curl -fsS http://localhost:8080/health
    ```
  </Step>
</Steps>

### Cargo Build Options

<AccordionGroup>
  <Accordion title="Feature Flags">
    - **`stateful`**: Enables Postgres key store via SQLx
    - **`chacha`**: Adds ChaCha20-Poly1305 algorithm support (optional)
    
    Example:
    ```bash
    cargo build --release --features stateful,chacha
    ```
  </Accordion>
  
  <Accordion title="Production Builds">
    Build optimized binaries:
    ```bash
    cargo build --release --bin qauth-api --features stateful
    ```
    
    Binary location:
    ```
    target/release/qauth-api
    ```
    
    Strip symbols for smaller size:
    ```bash
    strip target/release/qauth-api
    ```
  </Accordion>
</AccordionGroup>

## Environment Configuration

All QIMEM binaries read configuration from environment variables. Reference `.env.example`:

```bash .env.example
QIMEM_MODE=stateless
DATABASE_URL=postgres://postgres:postgres@postgres:5432/qimem
QIMEM_BIND=0.0.0.0:8080
RUST_LOG=info
```

### Configuration Reference

<ParamField path="QIMEM_MODE" type="string" default="stateless">
  Storage mode: `stateless` (in-memory) or `stateful` (Postgres)
</ParamField>

<ParamField path="DATABASE_URL" type="string" required>
  Postgres connection string (required when `QIMEM_MODE=stateful`)
  
  Format: `postgres://user:password@host:port/database`
</ParamField>

<ParamField path="QIMEM_BIND" type="string" default="0.0.0.0:8080">
  HTTP server bind address
  
  Examples:
  - `0.0.0.0:8080` - Listen on all interfaces
  - `127.0.0.1:8080` - Localhost only
  - `[::]:8080` - IPv6 all interfaces
</ParamField>

<ParamField path="RUST_LOG" type="string" default="info">
  Logging level using `tracing` crate
  
  Levels: `error`, `warn`, `info`, `debug`, `trace`
  
  Module-specific filtering:
  ```bash
  RUST_LOG=qimem=debug,sqlx=warn
  ```
</ParamField>

## Production Deployment

<Warning>
Production deployments require careful security configuration. Follow these best practices:
</Warning>

### Stateful Mode with Managed Postgres

<Steps>
  <Step title="Provision a managed Postgres instance">
    Use a managed service like AWS RDS, Google Cloud SQL, or Azure Database for PostgreSQL:
    
    - PostgreSQL 14 or later
    - Encrypted storage at rest
    - Automated backups enabled
    - SSL/TLS connections enforced
  </Step>

  <Step title="Configure DATABASE_URL with SSL">
    ```bash
    DATABASE_URL=postgres://user:pass@db.example.com:5432/qimem?sslmode=require
    ```
    
    For mutual TLS:
    ```bash
    DATABASE_URL=postgres://user:pass@db.example.com:5432/qimem?sslmode=verify-full&sslrootcert=/path/to/ca.crt
    ```
  </Step>

  <Step title="Run migrations">
    Migrations run automatically on startup. To run manually:
    ```bash
    sqlx migrate run --database-url $DATABASE_URL
    ```
  </Step>
</Steps>

### Security Hardening

<AccordionGroup>
  <Accordion title="Network Security">
    - Deploy behind a reverse proxy (nginx, Envoy, Cloudflare)
    - Enable TLS 1.3 with strong cipher suites
    - Use firewall rules to restrict database access
    - Implement rate limiting to prevent brute force
  </Accordion>
  
  <Accordion title="Key Material Protection">
    - QIMEM uses `zeroize::Zeroizing` for automatic memory cleanup
    - No key bytes are logged (enforced by `#![deny(unsafe_code)]`)
    - Consider hardware security modules (HSMs) for master keys
    - Rotate signing keys regularly via `/v1/auth/keys/rotate`
  </Accordion>
  
  <Accordion title="Monitoring and Observability">
    Health endpoints:
    - `GET /health` - Service liveness
    - `GET /ready` - Readiness (DB connectivity)
    - `GET /v1/security/health` - Security subsystem health
    
    Configure structured logging:
    ```bash
    RUST_LOG=qimem=info,tower_http=debug
    ```
    
    Integrate with metrics exporters (Prometheus, Datadog, etc.)
  </Accordion>
</AccordionGroup>

### Container Orchestration

<CodeGroup>
  ```yaml Kubernetes
  apiVersion: apps/v1
  kind: Deployment
  metadata:
    name: qauth-api
  spec:
    replicas: 3
    selector:
      matchLabels:
        app: qauth-api
    template:
      metadata:
        labels:
          app: qauth-api
      spec:
        containers:
        - name: qauth-api
          image: qimem/qauth-api:latest
          ports:
          - containerPort: 8080
          env:
          - name: QIMEM_MODE
            value: "stateful"
          - name: DATABASE_URL
            valueFrom:
              secretKeyRef:
                name: qimem-secrets
                key: database-url
          - name: RUST_LOG
            value: "info"
          livenessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 10
            periodSeconds: 10
          readinessProbe:
            httpGet:
              path: /ready
              port: 8080
            initialDelaySeconds: 5
            periodSeconds: 5
  ```
  
  ```yaml Docker Swarm
  version: '3.8'
  services:
    qauth-api:
      image: qimem/qauth-api:latest
      deploy:
        replicas: 3
        restart_policy:
          condition: on-failure
      environment:
        QIMEM_MODE: stateful
        DATABASE_URL: postgres://user:pass@postgres:5432/qimem?sslmode=require
        RUST_LOG: info
      ports:
        - "8080:8080"
      healthcheck:
        test: ["CMD", "curl", "-fsS", "http://localhost:8080/health"]
        interval: 10s
        timeout: 3s
        retries: 3
  ```
</CodeGroup>

## Health Checks

QIMEM provides multiple health check endpoints for different purposes:

<ResponseField name="GET /health" type="object">
  Basic liveness probe
  
  ```json
  {"status":"ok"}
  ```
  
  **Use case**: Kubernetes liveness probe, load balancer health check
</ResponseField>

<ResponseField name="GET /ready" type="object">
  Readiness probe with dependency checks
  
  ```json
  {"status":"ready"}
  ```
  
  **Use case**: Kubernetes readiness probe, confirms database connectivity in stateful mode
</ResponseField>

<ResponseField name="GET /v1/security/health" type="object">
  Security subsystem health
  
  ```json
  {"status":"ok"}
  ```
  
  **Use case**: Monitoring security-specific components
</ResponseField>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Connection refused on port 8080">
    **Cause**: Service not running or wrong bind address
    
    **Solution**:
    ```bash
    # Check if process is running
    ps aux | grep qimem
    
    # Verify QIMEM_BIND matches your request
    echo $QIMEM_BIND
    
    # Check Docker container logs
    docker compose logs qimem-api
    ```
  </Accordion>
  
  <Accordion title="Database connection failed">
    **Cause**: Invalid `DATABASE_URL` or Postgres not ready
    
    **Solution**:
    ```bash
    # Test Postgres connectivity
    psql $DATABASE_URL -c "SELECT 1;"
    
    # Check Docker Compose health
    docker compose ps
    
    # Verify DATABASE_URL format
    echo $DATABASE_URL
    ```
  </Accordion>
  
  <Accordion title="Build fails with stateful feature">
    **Cause**: Missing SQLx dependencies or database not accessible during build
    
    **Solution**:
    ```bash
    # SQLx compile-time verification needs DATABASE_URL
    export DATABASE_URL=postgres://postgres:postgres@localhost:5432/qimem
    
    # Or use offline mode
    cargo build --features stateful --offline
    ```
  </Accordion>
  
  <Accordion title="Permission denied errors in Docker">
    **Cause**: Container runs as non-root user `qimem` (UID 10001)
    
    **Solution**:
    Ensure mounted volumes have correct permissions:
    ```bash
    chown -R 10001:10001 /path/to/volume
    ```
  </Accordion>
</AccordionGroup>
